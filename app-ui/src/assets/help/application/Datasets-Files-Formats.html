The CSV file must include a header row containing the field (column) names, as in the example below. Use Windows line ending (CRLF) to end each line. Use double quotes to enclose field names that contain a comma, multiple words, line ending, or double quotes.

Sample CSV file:

"OrganisationId,WeekDay","Times","IsOpen","OpeningTimeType" "1186","Monday","09:30-13:00","True","General" "1186","Monday","13:30-17:30","True","General" ...

There are some online services (eg. Lint) that can help you validate your CSV files.

Here are some guidelines

  • The first row is a header row that contains the names of some columns.
  • The rest of the rows are data rows that contain a number of fields.
  • Line endings use CRLF (Windows line endings) and the column names and fields are separated by commas.
  • If a field contains a comma, a line ending or a double quote then the field is escaped by wrapping it in double quotes. Within a field that's escaped like that, any double quotes are doubled up.
