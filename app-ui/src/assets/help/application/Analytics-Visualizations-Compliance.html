Compliance checking can be done after a reference model is set.

A variant is considered non-compliant if it doesn't follow any of the variants that constitute the reference model. Only the paths are considered for compliance checking.

  • The level of non-compliance is shown as a number of variants and as a percentage of cases that aren't compliant with the reference set.
  • A table lists all non compliant variants, which can be sorted by columns.
  • A visual representation shows the differences between the reference model and a selected non-compliant variant.
  • The degree of non-compliance is shown as the difference between the variant (row) and the reference model.
  • Number of cases and duration metrics are shown.
