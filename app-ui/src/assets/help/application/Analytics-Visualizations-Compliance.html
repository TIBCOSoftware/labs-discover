Compliance checking can be done after a reference model is set.

A variant is qualified as non compliant if it doesn't follow any of the variants that constitute the reference model. Only the paths are considered for compliance checking.

What you can see at a glance is
  • The level of non compliance in the form of the number of variants, and % of cases that aren't compliant with the reference set.
  • A table showing all non compliant variants
  • A visual representation of the differences between the reference model and a selected non compliant variant

    • The table of non compliant variants can be sorted by any of these columns:

  • Degree of non compliance, expressed as the number of difference between the variant (row) and the reference model
  • Number of cases
  • Duration metrics

    • You can select a variant in the table and perform the following actions

  • Visualize the process compared to the reference model.
  • Mark the variant as checked so that next time you open the analysis you won't need to inspect it again.
  • Raise an investigation so that the non compliance can be escalated, validated and the root cause problem resolved so that it doesn't occur again in the future.
  • Track the status of the investigation
