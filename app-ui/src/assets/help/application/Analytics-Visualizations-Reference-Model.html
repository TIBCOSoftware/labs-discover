The reference model is the set of best practices or compliance rules to follow.

Here's how it works:

  • Initially the reference model is empty.
  • As you discover variants, you can Add one or more to the reference model.
  • The reference model is then persisted in the backend, therefore next time you open the analysis it will appear.
  • Once set, you can check compliance of the cases against that reference set.
  • At any time, you can add/remove variants from the reference model.

    • Setting the reference model does not require any modeling. Instead, it follows a discovery approach, expecting the data and cases to contain at least one occurrence of them.
