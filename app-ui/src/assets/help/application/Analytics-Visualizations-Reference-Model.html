The reference model is the set of best practices or compliance rules to follow.

Here's how it works:

  • Initially the reference model is empty.
  • As you discover variants, you can add them to the reference model.
  • The reference model persists in the backend and appears the next time you open the analysis.
  • You can check case compliance against the reference set.
  • You can add or remove variants from the reference model at any time.
